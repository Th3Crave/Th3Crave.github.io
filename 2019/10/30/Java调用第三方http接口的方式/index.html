Java调用第三方http接口的方式

2019-10-30 | technology
条评论

在实际开发过程中，我们经常需要调用对方提供的接口或测试自己写的接口是否合适。很多项目都会封装规定好本身项目的接口规范，所以大多数需要去调用对方提供的接口或第三方接口（短信、天气等）。

在Java项目中调用第三方接口的方式有：

  1. 通过JDK网络类Java.net.HttpURLConnection；
  2. 通过common封装好的HttpClient；
  3. 通过Apache封装好的CloseableHttpClient；
  4. 通过SpringBoot-RestTemplate；

通过JDK网络类java.net.HttpURLConnection

通过apache common封装好的HttpClient

httpClient的get或post请求方式步骤：

  1. 生成一个HttpClient对象并设置相应的参数；
  2. 生成一个GetMethod对象或PostMethod并设置响应的参数；
  3. 用HttpClient生成的对象来执行GetMethod生成的Get方法；
  4. 处理响应状态码；
  5. 若响应正常，处理HTTP响应内容；
  6. 释放连接。

通过Apache封装好的CloseableHttpClient

CloseableHttpClient是在HttpClient的基础上修改更新而来的，这里还涉及到请求头token的设置（请求验证），利用fastjson转换请求或返回结果字符串为json格式，当然上面两种方式也是可以设置请求头token、json的。
