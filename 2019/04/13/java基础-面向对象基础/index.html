面向过程与面向对象

面向过程：是分析出解决问题所需要的步骤，然后用函数将这些步骤一个一个实现，使用的时候一个一个依次调用这些函数。
面向对象：是把要解决的问题按照一定的规则（抽象）划分成多个独立的对象，然后通过调用这些对象的方法来解决问题。
OOA(面向对象的分析)：通常把构成问题域的事务分解成各个对象，给对象赋予相应的属性和行为，通过多个对象的相互协调来解决问题。建立对象的目的不是为了完成一个步骤，而是为了描述某个事物在整个解决问题的步骤中的行为。

面向对象的思想

1.分析系统中涉及的事务，根据“对象”抽象出“类”
2.找出对象共有的特点，并且在类中定义为属性
3.找出对象共有的行为，并且定义为类的方法

面向对象概念

抽象

对象

客观世界中真实存在的，某一类事物的某个个体。

类

对某一类事物的描述。类是一组具有相同属性和行为的对象的抽象。

类和对象的关系

抽象和具体的关系。对象是类的一个实例。

面向对象三大特性

1.封装

把数据和方法包装进类中，以及具体实现的隐藏，共同称作是封装。隐藏对象的属性和方法的实现细节，方便修改的实现，只能通过规定方法访问。

封装关键字

private：私有的，不对外公开，类的private成员只能被该类的成员访问。private权限最小，限制类外访问。对于父类的private成员变量，子类无法直接访问，只能通过public的get函数调用。
default：在类内以及在同一个包中的其它类可以访问。
protected：类内、同一个包中的其他类以及不同包的子类。
public：公有的。

2.继承(子类 is a kind of 父类)

子类自动共享父类的属性和方法的机制
父类：超类、原生类、基类
子类：派生类、孩儿类

(重要)在一个子类被创建的时候，首先会在内存中创建一个父类对象，然后在父类对象外部放上子类独有的属性，两者合起来形成一个子类的对象。

子类对象确实拥有父类对象中所有的属性和方法，但是父类对象中的私有属性和方法，子类是无法访问到的，只是拥有，但不能使用。就像有些东西你可能拥有，但是你并不能使用。

子类中的成员：
1）从父类继承下来的可视化(非private)的成员（字段、方法）
2）子类特有的成员（字段、方法、构造器）
3）重载(overload)父类的成员（方法）–多态的一种 编译时多态
4）覆盖(重写override)父类的成员（方法）–多态的一种 运行时多态
本质：替换
5）隐藏父类的成员（字段）

父类中不同访问权限修饰符的成员，在子类中的调用权限
1.包内
1)子类方法中可以直接访问父类非private成员
2)子类中通过父类对象可以访问父类非private成员
3)通过子类对象能够访问父类非private成员
4)父类中的private属性可以通过父类的非private方法获取
2.包外
1)包外子类只能访问父类的public和protected成员
2)包外子类通过父类对象只能访问父类public成员
3)包外子类通过子类对象只能访问父类public和protected成员
public > protected > default > private

重载的条件

a）发生在同一个类中或者父子类之间
b）方法名称相同，参数列表（参数的数据类型、参数的个数、参数的顺序）不同
c）返回值无所谓（返回值相同可以，也可以不相同）
d）访问权限无所谓

构造器重载：
    this两种用法：
    this.成员(字段、方法):调用本类的其他的字段或者方法
    this（参数列表）:调用本类的其他构造器，这句话必须位于构造器的第一句

覆盖的条件

a）发生在父子类之间
b）方法名称相同，参数列表（参数的数据类型、参数的个数、参数的顺序）相同——方法签名必须相同
c）返回值必须相同
d）子类覆盖方法的访问权限大于等于父类被覆盖方法的访问权限
e）子类不能抛出比父类异常类更大的异常（运行时异常例外）

super两种用法:父类的
super.成员（字段、方法）：调用父类的字段或者方法        
super(参数列表)：调用父类的构造器,这句话必须位于子类构造器的第一句
如果父类中没有无参的构造器，子类必须在构造器中显式的调用super(参数列表)或者this(参数列表)，否则编译错误
如果父类中有无参的构造器，子类中可以不需要调用super(参数列表),因为系统默认调用super()
系统默认调用super();
子类实例创建时构造器的调用顺序：
先调用父类构造器，然后调用子类构造器
java中类实例的创建过程(执行顺序)：父类的静态->子类的静态->父类的初始化块->父类的构造方法->子类的初始化块->子类的构造方法
Object： 所有引用类型的根

3.多态 Polymorphism

面试常问：对多态的理解！

多态对象！！！ 父类 变量 = new 子类();

父类引用指向子类对象

多态对象：
1.对于方法：覆盖父类方法，调用的是子类的方法。
2.对于属性(字段)：隐藏父类的属性，访问的还是父类的属性。因此，多态对象不能直接调用子类所特有的成员。

多态对象的三个必要条件：

1）要有继承或者实现
2）重写
3）父类的引用指向子类对象——在执行期间(动态绑定在执行期间，静态绑定在编译期间)判断所引用对象的实际类型，根据其实际类型调用相应的方法。

引用数据类型转换(继承关系中)

前提：具有继承或者实现关系(接口引用或者抽象类引用可以指向它们的子类对象)
上转型： 子——>父，小转大，自动转换。向上转换损失了子类扩展的属性和方法，仅可以使用从父类继承的属性和方法。
向下造型：父——>子，大转小，强制转换。特别注意：只有曾经向上转换过的对象才能向下转换，对象不允许不经过上转型而直接向下造型。

java中类实例的创建过程(执行顺序)：父类的静态->子类的静态->父类的初始化块->父类的构造方法->子类的初始化块->子类的构造方法

静态：static关键字

静态的，类成员，代码一加载到内存就执行。
静态资源的访问：类名.成员

static修饰属性

只能修饰属性，不能修饰局部变量！
所有对象共享，通常是常量，称为静态变量或类变量。不用static修饰的属性称为实例变量。
静态变量被所有对象共享，在内存中只有一个副本，JVM的静态方法区；非静态变量是对象拥有的，在创建对象时被初始化，JVM的堆。

static修饰方法

不需要实例化，可以直接访问，称为静态方法或者类方法。
static修饰的属性是类的属性，this代表一个对象，static修饰的属性不需要对象就能访问。
1.静态方法可以直接访问静态的成员，但是不能访问非静态的成员，如果要访问，需要创建对象实例来访问非静态的成员。
2.非静态方法可以直接访问非静态成员和静态成员。
3.静态方法中不能使用this或者super关键字

static修饰语句块 静态代码块

在内存中只执行一次，一加载到内存就执行。静态块经常用来进行类属性的初始化。

static修饰内部类 静态内部类

单例模式

保证一个类仅有一个实例，并提供一个访问它的全局访问点。

单例模式要点：
1.某个类只能有一个实例
2.它必须自行创建这个实例
3.必须自行向整个系统提供这个实例

单例模式实现

1.拥有一个私有构造器——使得无法在外部实例化一个对象
2.提供一个自身静态私有的成员变量——创建一个实例
3.提供一个公有的静态方法——返回该实例

final

final：最终的，终态的，一般会与static合在一起使用
变量(成员变量或者局部变量)：常量，不能第二次赋值，必须有初始值，在构造器中或者声明时赋初始值
方法：不能被重写
类：不能被继承

abstract

abstract：抽象的
抽象方法：只有方法的声明，没有方法体，抽象方法不能用static修饰
抽象类：抽象方法必须在抽象类中
抽象类的一些细节：
1）抽象方法必须放在抽象类中，但是抽象类中可以没有抽象方法
2）抽象类中可以有抽象方法和非抽象方法
3）抽象类中可以有字段
4）抽象类没有实例
5）抽象类有构造器，不是用来产生本抽象类的实例，而是产生子类实例时(需要先调用父类的构造器)，间接调用该抽象类的构造器
6）抽象类可以有父类，父类可以是非抽象的，也可以是抽象的
7）子类重写抽象类中的部分抽象方法，该子类仍然是抽象类
8）abstract不能和private、final、static一起使用。抽象类中abstract只能用public、protected访问权限修饰符修饰。
一个类必须为抽象类的情况：
1）当一个类的一个或多个方法是抽象方法时
2）当类是一个抽象类的子类，并且没有实现父类的所有抽象方法，即只实现部分抽象方法
3）当一个类实现一个接口，并且不能为全部抽象方法都提供实现时

interface

interface：接口，引用类型
规范、标准
弥补了单继承的不足，模拟实现多继承
成员的访问权限：public，接口中只能是public，默认就是public！！
接口与类之间：实现关系，implements
类单继承类，多实现接口
接口多继承接口，无法实现接口
没有实例
成员：常量和抽象方法
jdk1.8以上 常量和抽象方法，还有默认方法和静态方法
int i=10;//public static final int i=10;
void ma();//public abstract void ma(); 默认修饰字

抽象类和接口的区别

抽象类一般用于基础类的构建，能够用来限定交互接口。
1）抽象类中可以有变量或者常量，接口只能是常量(默认是static final的)
2）抽象类有构造器，接口没有构造器
3）抽象类中可以有抽象方法或者非抽象方法，接口中有抽象方法或者默认方法
4）类之间是单继承的，类与接口之间是多实现，接口与接口之间是多继承
5）无关的类不可能有同一个抽象父类，但是无关的类可以实现同一个接口

内部类

在一个类的内部再声明一个类
好处：
1）对外隐藏了外部类的实现细节
2）对于内部类来说可以访问所有的外部类的成员
3）内部类可以模拟实现多继承（接口或者内部类）
4）当接口与父类中的某个方法同名时，可以通过内部类解决
分类：
成员内部类
静态内部类
局部内部类
匿名内部类

成员内部类：

1）定义在方法外部，并且没有static修饰的类称为成员内部类
2）在外部类中可以直接创建内部类对象
3）在其他的类中如果需要创建内部类对象，必须先创建外部类的对象的实例，
然后在外部类对象实例的基础上创建内部类对象
外部类.内部类 变量名=外部类对象名.new 内部类();
4)内部类可以直接访问外部类的所有成员，
如果外部类想要访问内部类成员，必须先创建内部类对象，通过实例访问
5）在内部类中不能定义静态成员，只能定义实例成员
6）如果内部类的成员与局部变量以及外部类的成员同名
外部类名.this.成员名 表示外部类的成员
this.成员 表示内部类的成员

静态内部类：

1）定义在方法的外部，用static修饰的类称为静态内部类
2）在外部类中可以直接创建内部类对象
3）在其他的类中如果需要创建内部类对象，可以直接new创建即可，不需要通过外部类的实例
外部类名.内部类名 变量名=new 外部类名.内部类名（）；
4）内部类可以直接访问外部类的静态成员，如果需要访问非静态的成员需要产生实例
如果外部类想要访问内部类成员，静态成员，类名.成员名访问；非静态的成员，产生内部类实例访问
5）静态内部类中可以定义静态成员和非静态的成员
6）在其他类中可以通过完整的路径名访问静态内部类中的静态成员。

局部内部类：

1）在局部代码块(方法、某个循环中、某个分支语句中)中定义的内部类称为局部内部类
2）局部内部类的成员只能在该方法内部可见，方法外无法使用
3）局部内部类与局部变量类似，不能使用private、protected、public、static修饰
4）局部内部类中不能包含静态成员
5）局部内部类可以直接访问外部类的所有成员和方法内的参数和变量

匿名内部类：

多态对象： 父类 变量名=new 子类（）；
匿名内部类： 父类 变量名=new 父类父接口{//匿名类的成员};
