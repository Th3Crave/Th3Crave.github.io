mysql是关系型数据库。关系型数据库采用二维表结构组织和管理数据，并规定了表内和表间数据的依赖关系。关系数据库是指一些相关的表和其他数据库对象的集合。对于关系数据库来说，关系就是表的同义词。关系型数据库管理系统以表、字段和记录等结构来组织数据，表用来保存数据，每个表由一组字段来定义其结构，记录是表中的一条数据。

2MySQL 执行引擎

一个库中可以有多个不同存储引擎的表。
innoDB，MyISAM
InnoDB采用聚集索引(聚簇索引)，若一个主键被定义，该主键则作为密集索引；若没有主键被定义，该表的第一个唯一非空索引则作为密集索引；若不满足以上条件，innoDB内部会生成一个隐藏主键(rowid，密集索引，该主键是一个六字节的列，该列的值随着记录的插入而自增)。
即innoDB必须得有一个主键作为密集索引存在，innodb引擎里的主键一般都是聚集索引；非主键索引都是非聚集索引，非主键索引存储相关键位和其对应的主键值，要进行两次查找(先通过非主键索引去查找主键索引，再通过主键索引去查找数据的物理地址)。
MyIsam采用稀疏索引(非聚簇索引)。MyISAM中索引一个文件，数据一个文件，主键索引和辅助索引是平等的，不需要像innoDB的辅助索引一样要进行两次查找。
InnoDB和MyISAM数据库的表的结构定义信息都存储在frm文件中；InnoDB的数据和索引存储在ibd文件中，MyISAM索引存在MYI文件中，数据存储在MYD文件中。

5索引

为什么mysql中给某一个字段添加一个唯一性约束，会自动生成一个索引？
唯一性约束是为了做到表中数据不能有重复值，但是数据库是如何保证数据没有重复值呢？要在存储数据的时候查询一遍看是否重复。那怎样查询最快呢？mysql中创建索引查询最快，所以在创建唯一性约束时就创建了索引。

0)索引定义？
在数据库层面，索引是为了加快对表中数据行的检索而创建的一种分散存储的数据结构。

1)为什么要使用索引
当表的数据量比较大时，查询操作会比较耗时，为避免全表扫描的情况发生，建立索引是加快查询速度的有效手段，索引的灵感来自字典。索引虽然能够加快数据库查询，但需要占用一定的存储空间，当基本表更新时，索引要进行相应的维护。
全表扫描：在表中一条数据一条数据进行搜索。

2)什么样的信息能成为索引
主键、唯一键等能让数据具有一定区分性的字段。

3)索引的数据结构
主流是B+树，还有hash索引和位图索引。
1.生成索引，建立二叉查找树进行二分查找；当用自增的主键做索引时，二叉树查找树会生成一条链表，时间复杂度从O(logn)将为O(n)。平衡二叉查找树是为了解决二叉查找树退化成链表的问题。
二叉树缺陷：二叉树深度太大，搜索时IO次数过多，节点数据内容太少。
2.生成索引，建立B-Tree结构进行查找；

3.生成索引，建立B+-Tree结构进行查找：支持利用组合索引中的部分索引来查询；

B+树节点关键字搜索采用闭合区间(左闭合，关键字和路数一样多)；B+树非叶子节点不保存数据相关信息，只保存关键字和子节点的引用；B+树关键字对应的数据保存在叶子节点中；B+树叶子节点是顺序排列的，并且相邻节点具有顺序引用的关系。
mysql选用B+树，B+树是B树的变种多路绝对平衡查找树，具有B树的优势；扫库、表能力更强；磁盘读写能力更强；排序能力更强；查询效率相对稳定(B+树有多高，就要IO多少次，因为B+树中只有叶子节点中存储了数据区域的引用，其他非终端节点只存储了关键字和指向子节点的指针)。

4.生成索引，建立Hash结构进行查找：建立若干个桶，将索引属性按照其散列函数值映射到相应桶中，桶中存放索引属性值和相应的元组指针。仅仅能满足“=”，“IN”，不能使用范围查询；无法被用来避免数据的排序操作；不能利用部分索引键查询；不能避免表扫描；遇到大量Hash值相等的情况后性能并不一定就会比B树索引高；
5.BitMap位图索引，用位向量记录索引属性中可能出现的值，每个位向量对应一个可能值。

4)密集索引和稀疏索引的区别
密集索引文件中的每个搜索码值都对应一个索引值，数据库表行中数据的物理顺序与键值的索引顺序相同。
稀疏索引文件只为索引码的某些值建立索引项。

5)联合索引

联合索引的选择：最左匹配选择>离散度高原则>最少空间原则

6)覆盖索引
如果查询的列，通过索引项的信息可以直接返回，则该索引称之为查询sql的覆盖索引。(查询的信息就在索引的关键字中，通过索引项的信息可以直接返回，就不需要再去数据区加载相关内容。)

Hash 索引、B 树索引（B+树、和B树、R树）
普通索引、唯一索引
覆盖索引、最左前缀原则、索引下推

索引的几大原则

1.列的离散性。计算公式：count(不同的行数)：count(总col)，离散性越高，选择性越好。
2.最左匹配原则。对索引中关键字进行计算(对比)，一定是从左往右依次进行，且不可跳过。
3.联合索引
联合索引中如果不是按照索引最左列开始查找，无法使用索引；精确匹配最左前列并范围匹配另外一列可以用到索引；如果查询中有某个列的范围查询，则其右边的所有列都无法使用索引。

1sql语言

DDL:数据库定义语句，用来创建数据库中的表、索引、视图、存储过程、触发器等
CREATE,ALTER,DROP,TRUNCATE,COMMENT,RENAME
DML:数据操纵语句，用来查询、添加、更新、删除等
SELECT,INSERT,UPDATE,DELETE,MERGE,CALL,EXPLAIN PLAN,LOCK TABLE
DCL:数据控制语句，用于授权/撤销数据库及其字段的权限
GRANT,REVOKE。
TCL:事务控制语句，用于控制事务
COMMIT,ROLLBACK,SAVEPOINT,SET TRANSACTION

INSERT:应该在插入单行的时候使用VALUES，在插入多行的时候使用VALUE
DELETE和TRUNCATE(截断):两者都可以删除表中的记录。区别是按条件删除使用delete，TRUNCATE属于DDL，释放了存储空间，不能回滚，DELETE属于DML，删除时不释放存储空间，可以进行回滚。
SAVEPOINT:在此关键字所在处，设置一个保存点，可以使用ROLLBACK TO回滚到此保存点，而不回滚整个事务。

2mysql CHECK无效

不同于SQL，在MYSQL中，CHECK只是一段可调用但无意义的子句。MySQL会直接忽略。
CHECK子句会被分析，但是会被忽略。
请参见“CREATE TABLE语法”：接受这些子句但又忽略子句的原因是为了提高兼容性，以便更容易地从其它SQL服务器中导入代码，并运行应用程序，创建带参考数据的表。
1.如果需要设置CHECK约束的字段范围小，并且比较容易列举全部的值，就可以考虑将该字段的类型设置为枚举类型 enum()或集合类型set()。比如性别字段可以这样设置，插入枚举值以外值的操作将不被允许。

3事务

事务是一组原子性的SQL查询，或者说mysql的一个独立的工作单元。如果数据库引擎能够成功地对数据库应用该组查询的全部语句，那么就执行该组查询；如果其中有任何一条语句因为崩溃或其他原因无法执行，那么所有的语句都不会执行，任何一个步骤失败，必须回滚所有的步骤。事务内的语句，要么全部执行成功，要不一句也不执行。
事物的ACID特性，原子性atomicity、一致性consistency、隔离性isolation、持久性durability。
mysql设置事务手动提交: SET @@autocommit = 0; SHOW VARIABLES LIKE ‘autocommit’;

5LIMIT

mysql分页查询 用LIMIT来限制记录的行数
1)SELECT 子句 LIMIT n：表示从查询结果中取前n条记录返回
2)SELECT 自居 LIMIT index,length：表示从查询结果中的第index条记录开始返回length条。其中，index是select记录的索引，第一条记录的索引是0，length表示总共取几条。
