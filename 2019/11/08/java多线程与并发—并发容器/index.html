Map

日常的开发过程中，Map集合使用率是比较高的容器，像hashMap(hash结构)，treeMap(tree结构)，linkedHashMap(hash结构，加双向链表)等，但上述这些容器都不是并发容器，其中的方法都不能保证原子性，而在高并发编程中，我们使用更多的是hashTable(hash结构)，Collection.synchronizedMap(将非线程安全的容器变为线程安全的容器)，concurrentHashMap(hash结构)，concurrentSkiplistMap(跳表结构)。

  • 非并发业务不需要排序：hashMap
  • 非并发业务需要排序：treeMap，linkedHashMap
  • 并发业务量少不需要排序：hashTable，Collection.synchronizedMap
  • 并发业务量大不需要排序：concurrentHashMap
  • 并发业务量大需要排序：concurrentSkiplistMap

List

  • 无并发问题，读多：ArrayList
  • 无并发问题，写多：LinkedList
  • 有并发问题，读多写少：CopyOnWriteList(写时复制容器，插入时复制新的List装新的集合)
  • 有并发问题，写多：Vector，Collection.synchronizedList

ConcurrentLinkedQueue

队列在并发编程中使用的频率是非常高的。
ConcurrentLinkedQueue为单向链表结构，内部加锁。

BlockingQueue

阻塞式队列，当队列中没值时，消费者自动等待，当队列满时，生产者自动等待。

  • LinkedBlockingQueue
  • ArrayBlockingQueue

DelayQueue

TransferQueue
